DOCUMENTACAO DO CONTROLLER PIONEER3-AT COM GYMNASIUM E STABLE-BASELINES3
=======================================================================

Este arquivo descreve a organizacao dos codigos do controller localizado em:

Robot_model/controllers/base_controller/

A ideia geral da refatoracao foi separar responsabilidades:

- config.py: constantes gerais do projeto.
- actions.py: enum com as acoes discretas do robo.
- devices.py: acesso aos sensores e motores do Webots.
- rewards.py: funcao de recompensa.
- environment.py: ambiente Gymnasium.
- base_controller.py: ponto de entrada do Webots, criacao do ambiente, modelo e treino.


ARQUIVO: config.py
==================

Objetivo
--------
Centralizar constantes usadas pelo projeto para evitar numeros magicos espalhados
pelo codigo.

Constantes
----------

HISTORY = 5
    Quantidade maxima de leituras guardadas por sensor para calcular a media movel.

NUM_SENSORS = 16
    Numero de sensores de distancia usados pelo Pioneer3-AT.
    O codigo busca sensores de "so0" ate "so15".

OBSERVATION_SIZE = NUM_SENSORS + 4 + 4
    Tamanho total da observacao entregue ao agente.
    A observacao concatena:
    - 16 leituras normalizadas dos sensores;
    - 4 velocidades comandadas e normalizadas das rodas;
    - 4 valores one-hot representando a ultima acao aplicada.

FORWARD_SPEED = 3.0
    Velocidade aplicada aos quatro motores quando a acao e andar para frente.

TURN_SPEED = 4.0
    Velocidade usada nas rodas para girar o robo para esquerda ou direita.

SENSOR_LIMIT = 1024.0
    Valor usado para normalizar as leituras dos sensores.
    Uma leitura bruta e dividida por este valor e depois limitada ao intervalo
    entre 0.0 e 1.0.

COLLISION_THRESHOLD = 0.8
    Limite frontal usado para considerar colisao ou perigo critico.
    Se o perigo frontal for maior ou igual a este limite, o ambiente considera
    que houve colisao.

SIDE_COLLISION_THRESHOLD = 0.98
    Limite geral usado para detectar proximidade extrema em qualquer direcao.
    Permite seguir ao lado de paredes sem encerrar o episodio cedo demais.

MAX_STEPS = 1000
    Numero maximo de steps por episodio.
    Quando current_step chega a este valor, o episodio e truncado.

FRONT_SENSOR_IDS = (2, 3, 4, 5)
    Indices dos sensores usados para medir perigo frontal.

LEFT_SENSOR_IDS = (0, 1, 2)
    Indices dos sensores usados para medir perigo do lado esquerdo.

RIGHT_SENSOR_IDS = (5, 6, 7)
    Indices dos sensores usados para medir perigo do lado direito.

RANDOM_SPAWN = True
    Se True, cada episodio comeca em uma posicao e orientacao aleatorias.

SPAWN_X_RANGE = (-1.25, 1.25)
SPAWN_Y_RANGE = (-1.25, 1.25)
    Intervalos usados para sortear a posicao no plano da arena.
    Mantem margem para o corpo do robo dentro da arena 5 x 5 e reduz resets
    apontados diretamente para paredes.

SPAWN_YAW_RANGE = (-3.14159, 3.14159)
    Intervalo em radianos usado para sortear a orientacao em torno do eixo Z.

SPAWN_MAX_START_DANGER = 0.45
    Maior perigo frontal aceito no inicio do episodio.

SPAWN_MAX_ATTEMPTS = 20
    Quantidade maxima de tentativas para encontrar um spawn seguro.

Como usar
---------
Importe as constantes onde forem necessarias:

    from config import MAX_STEPS, SENSOR_LIMIT, COLLISION_THRESHOLD, SIDE_COLLISION_THRESHOLD


ARQUIVO: actions.py
===================

Objetivo
--------
Definir nomes legiveis para as acoes discretas do robo.

Classe: Action(IntEnum)
----------------------
Enum inteiro usado pelo ambiente, pelas rodas e pela funcao de recompensa.
Como herda de IntEnum, pode ser convertido para int e tambem pode receber um int.

Valores
-------

Action.STOP = 0
    Para o robo.

Action.FORWARD = 1
    Move o robo para frente.

Action.TURN_LEFT = 2
    Gira o robo para a esquerda.

Action.TURN_RIGHT = 3
    Gira o robo para a direita.

Como usar
---------

    from actions import Action

    action = Action(1)
    if action == Action.FORWARD:
        ...

Retorno
-------
A classe em si nao tem uma funcao de retorno. Ela fornece constantes nomeadas
que tambem funcionam como inteiros.

Classe: DistanceSensors
----------------------
Responsavel por buscar, habilitar, ler e suavizar as leituras dos sensores de
distancia do Pioneer3-AT.

Metodo: __init__(self, robot, timestep)
---------------------------------------
Argumentos:
    robot:
        Objeto do Webots que deve fornecer getDevice(nome).
        Normalmente e o Supervisor ou Robot criado no controller.

    timestep:
        Timestep usado para habilitar os sensores.
        Normalmente vem de robot.getBasicTimeStep().

O que faz:
    - Cria a lista de nomes "so0" ate "so15".
    - Busca cada sensor com robot.getDevice(nome).
    - Habilita cada sensor com sensor.enable(timestep).
    - Prepara listas internas para historico e medias.

Retorno:
    Nao retorna valor. Inicializa o objeto.

Atributos principais:
    self.sensors:
        Lista com os sensores do Webots.

    self.names:
        Lista com os nomes dos sensores.

    self.raw_values:
        Lista de listas com o historico recente de cada sensor.

    self.avg_values:
        Lista com a media movel atual de cada sensor.


Metodo: reset_history(self)
---------------------------
Argumentos:
    Nenhum.

O que faz:
    Limpa o historico de leituras dos sensores e limpa as medias.
    Deve ser chamado no reset do ambiente para iniciar um episodio sem leituras
    antigas influenciando a observacao inicial.

Retorno:
    Nao retorna valor.


Metodo: read_all(self)
----------------------
Argumentos:
    Nenhum.

O que faz:
    - Le todos os sensores com getValue().
    - Adiciona a leitura ao historico de cada sensor.
    - Mantem no maximo HISTORY leituras por sensor.
    - Calcula a media movel de cada sensor.

Retorno:
    Lista de floats com NUM_SENSORS valores medios.
    Exemplo conceitual:

        [12.5, 9.0, 150.2, ..., 30.1]

Como usar:

    sensors = DistanceSensors(robot, timestep)
    values = sensors.read_all()


Classe: Wheels
--------------
Responsavel por buscar os motores do Pioneer3-AT e aplicar velocidades de acordo
com uma acao discreta.

Metodo: __init__(self, robot)
-----------------------------
Argumentos:
    robot:
        Objeto do Webots que deve fornecer getDevice(nome).

O que faz:
    - Busca os quatro motores:
        "front left wheel"
        "front right wheel"
        "back left wheel"
        "back right wheel"
    - Coloca cada motor em modo velocidade com setPosition(float("inf")).
    - Inicializa cada motor parado com setVelocity(0).
    - Prepara uma lista com as velocidades comandadas para as quatro rodas.
    - Prepara uma lista one-hot com a ultima acao aplicada.

Retorno:
    Nao retorna valor. Inicializa o objeto.


Metodo: set_speeds(self, front_left, front_right, back_left, back_right)
-----------------------------------------------------------------------
Argumentos:
    front_left, front_right, back_left, back_right:
        Velocidades aplicadas individualmente aos quatro motores.

O que faz:
    Atualiza as velocidades comandadas guardadas em self.speeds e chama
    setVelocity() em cada motor.

Retorno:
    Nao retorna valor.


Metodo: stop(self)
------------------
Argumentos:
    Nenhum.

O que faz:
    Define velocidade 0 para todos os motores.

Retorno:
    Nao retorna valor.


Metodo: forward(self)
---------------------
Argumentos:
    Nenhum.

O que faz:
    Define FORWARD_SPEED para todos os motores.

Retorno:
    Nao retorna valor.


Metodo: turn_left(self)
-----------------------
Argumentos:
    Nenhum.

O que faz:
    Gira o robo para a esquerda.
    As rodas esquerdas recebem velocidade negativa e as direitas recebem
    velocidade positiva.

Retorno:
    Nao retorna valor.


Metodo: turn_right(self)
------------------------
Argumentos:
    Nenhum.

O que faz:
    Gira o robo para a direita.
    As rodas esquerdas recebem velocidade positiva e as direitas recebem
    velocidade negativa.

Retorno:
    Nao retorna valor.


Metodo: action(self, action)
----------------------------
Argumentos:
    action:
        Pode ser um int ou um Action.
        Valores aceitos:
            0 ou Action.STOP
            1 ou Action.FORWARD
            2 ou Action.TURN_LEFT
            3 ou Action.TURN_RIGHT

O que faz:
    Converte o argumento para Action e chama internamente:
        stop()
        forward()
        turn_left()
        turn_right()
    Tambem atualiza self.actions com a representacao one-hot da ultima acao.

Retorno:
    Nao retorna valor.

Como usar:

    wheels = Wheels(robot)
    wheels.action(Action.FORWARD)
    wheels.action(0)


Metodo: get_normalized_speeds(self)
-----------------------------------
Argumentos:
    Nenhum.

O que faz:
    Divide cada velocidade comandada pelo maior valor absoluto configurado
    entre FORWARD_SPEED e TURN_SPEED.

Retorno:
    Lista com quatro floats no intervalo entre -1.0 e 1.0.


Metodo: get_last_action(self)
-----------------------------
Argumentos:
    Nenhum.

O que faz:
    Retorna a representacao one-hot da ultima acao aplicada.

Retorno:
    Lista com quatro floats na ordem:

        [STOP, FORWARD, TURN_LEFT, TURN_RIGHT]


ARQUIVO: rewards.py
===================

Objetivo
--------
Guardar a funcao de recompensa separada da logica do ambiente.


Funcao: split_observation(observation)
--------------------------------------
Assinatura:

    split_observation(observation)

Argumentos:
    observation:
        Vetor completo retornado pelo ambiente.

O que faz:
    Separa a observacao em tres partes:
    - sensores normalizados;
    - velocidades comandadas e normalizadas das rodas;
    - ultima acao em formato one-hot.
    A recompensa usa sensores e velocidades. A acao one-hot permanece
    disponivel na observacao entregue ao agente, mas nao precisa ser usada pela
    recompensa porque a acao atual tambem e recebida como argumento.

Retorno:
    Tupla:

        sensor_obs, speed_obs, action_obs


Funcao: get_sensor_metrics(sensor_obs)
-------------------------------------
Assinatura:

    get_sensor_metrics(sensor_obs)

Argumentos:
    sensor_obs:
        Vetor com as leituras normalizadas dos sensores.

O que faz:
    Calcula metricas complementares para diferenciar um pico isolado de uma
    regiao ocupada por obstaculos:
    - media geral dos sensores;
    - medias frontal, esquerda e direita;
    - proporcao de sensores com valor maior ou igual a 0.5;
    - proporcao de sensores com valor maior ou igual a 0.75.

Retorno:
    Dicionario com as metricas calculadas.


Funcao: reward(action, observation, dangers, collision, distance_moved)
-----------------------------------------------------------------------
Assinatura:

    reward(action, observation, dangers, collision, distance_moved)

Argumentos:
    action:
        Int ou Action representando a acao executada.

    observation:
        Vetor completo com sensores, velocidades comandadas das rodas e ultima acao.

    dangers:
        Dicionario com as chaves "max", "front", "left" e "right".

    collision:
        Booleano indicando se houve colisao.

    distance_moved:
        Distancia realmente percorrida pelo robo no ultimo step.

O que faz:
    Implementa a recompensa usada pelo ambiente:
    - colisao retorna -50.0;
    - sem colisao comeca com recompensa base positiva 0.05;
    - deslocamento fisico real recebe recompensa;
    - perigo frontal reduz a recompensa;
    - proximidade critica em qualquer direcao recebe penalidade adicional;
    - andar para frente e bastante recompensado quando a frente esta livre;
    - seguir ao lado de uma parede sem obstaculo frontal recebe pequeno bonus;
    - virar e recompensado quando existe obstaculo frontal;
    - virar para o lado mais seguro recebe bonus extra baseado em picos e medias
      apenas quando existe obstaculo frontal;
    - andar para frente recebe incentivo adicional quando o comando realmente
      representa movimento frontal.

Retorno:
    Float com a recompensa calculada.

Como usar:

    result = reward(Action.TURN_LEFT, observation, dangers, collision, distance_moved)


ARQUIVO: environment.py
=======================

Objetivo
--------
Definir o ambiente Gymnasium do Pioneer3-AT.
Este arquivo deve cuidar do contrato do Gymnasium:

    reset() -> observation, info
    step(action) -> observation, reward, terminated, truncated, info

Ele tambem centraliza:
    - leitura e normalizacao das observacoes;
    - calculo dos perigos;
    - aplicacao de acoes;
    - reset da pose do robo usando Supervisor;
    - controle de termino/truncamento do episodio.


Classe: PioneerEnv(gym.Env)
--------------------------
Ambiente usado para treinamento e avaliacao.

Atributos importantes:
    action_space:
        spaces.Discrete(len(Action)).
        Atualmente equivale a Discrete(4).

    observation_space:
        spaces.Box(low=-1.0, high=1.0, shape=(OBSERVATION_SIZE,), dtype=np.float32).
        Atualmente equivale a Box de 24 valores:
        - 16 sensores normalizados entre 0.0 e 1.0;
        - 4 velocidades comandadas e normalizadas entre -1.0 e 1.0;
        - 4 valores one-hot da ultima acao.

    metadata:
        {"render_modes": []}


Metodo: __init__(self, robot, timestep=None, max_steps=MAX_STEPS,
                 sensor_limit=SENSOR_LIMIT,
                 collision_threshold=COLLISION_THRESHOLD,
                 side_collision_threshold=SIDE_COLLISION_THRESHOLD)
-----------------------------------------------------------------
Argumentos:
    robot:
        Objeto Supervisor ou Robot do Webots.
        Deve fornecer:
            getBasicTimeStep()
            getDevice(nome)
            step(timestep)
        Se for Supervisor, tambem pode fornecer:
            getSelf()
            simulationResetPhysics()

    timestep:
        Timestep do Webots.
        Se for None, usa robot.getBasicTimeStep().

    max_steps:
        Numero maximo de steps por episodio.

    sensor_limit:
        Valor usado para normalizar sensores.

    collision_threshold:
        Limite frontal normalizado usado para detectar colisao.

    side_collision_threshold:
        Limite normalizado usado para proximidade extrema em qualquer direcao.

O que faz:
    - Salva parametros do ambiente.
    - Cria DistanceSensors.
    - Cria Wheels.
    - Se possivel, salva translation e rotation iniciais via Supervisor.
    - Define action_space.
    - Define observation_space.

Retorno:
    Nao retorna valor. Inicializa o ambiente.


Metodo: reset(self, seed=None, options=None)
--------------------------------------------
Argumentos:
    seed:
        Seed opcional do Gymnasium.

    options:
        Argumento opcional do Gymnasium.
        Atualmente nao e usado diretamente.

O que faz:
    - Chama super().reset(seed=seed).
    - Zera current_step.
    - Para o robo.
    - Restaura a pose inicial ou sorteia uma pose segura se estiver usando Supervisor.
    - Limpa historico dos sensores.
    - Executa um step no Webots para atualizar sensores.
    - Rejeita spawns com perigo frontal alto ou colisao imediata.
    - Se nao encontrar spawn seguro, volta para a pose inicial.
    - Le a observacao inicial.
    - Calcula informacoes basicas.

Retorno:
    Tupla:

        observation, info

    observation:
        np.ndarray com shape (OBSERVATION_SIZE,), dtype np.float32.
        Atualmente possui 24 valores entre -1.0 e 1.0.

    info:
        Dicionario com:
            "max_sensor_normalized": maior valor normalizado dos sensores.
            "step": step atual, normalmente 0 no reset.
            "spawn_attempts": numero de tentativas usadas para iniciar o episodio.


Metodo: step(self, action)
--------------------------
Argumentos:
    action:
        Acao discreta aceita pelo action_space.
        Pode ser int 0, 1, 2, 3 ou Action equivalente.

O que faz:
    - Verifica se a acao pertence ao action_space.
    - Converte a acao para Action.
    - Incrementa current_step.
    - Aplica a acao nas rodas.
    - Avanca a simulacao do Webots com robot.step(timestep).
    - Le e normaliza a observacao.
    - Calcula dangers.
    - Detecta colisao.
    - Calcula recompensa chamando _calculate_reward().
    - Define terminated e truncated.
    - Monta info com _make_info().

Retorno:
    Tupla no formato Gymnasium:

        observation, reward, terminated, truncated, info

    observation:
        np.ndarray com shape (OBSERVATION_SIZE,).
        Sensores ficam entre 0.0 e 1.0; velocidades comandadas ficam entre
        -1.0 e 1.0; a ultima acao usa representacao one-hot.

    reward:
        Float com recompensa calculada.

    terminated:
        Booleano. True quando houve colisao.

    truncated:
        Booleano. True quando atingiu max_steps ou quando Webots retornou -1.

    info:
        Dicionario com:
            "collision": bool
            "action": int
            "step": int
            "max_sensor": float
            "front_danger": float
            "left_danger": float
            "right_danger": float
            "simulation_ok": bool
            "distance_moved": float
            "total_distance": float


Metodo: _calculate_reward(self, action, observation, dangers, collision,
                          distance_moved)
-----------------------------------------------------------
Argumentos:
    action:
        Action executada.

    observation:
        Vetor completo retornado por _get_observation().

    dangers:
        Dicionario retornado por _get_dangers().

    collision:
        Booleano indicando colisao.

    distance_moved:
        Distancia percorrida pelo robo no ultimo step.

O que faz:
    Chama reward(action, observation, dangers, collision, distance_moved).

Retorno:
    Float com a recompensa.


Metodo: _make_info(self, action, dangers, collision, simulation_ok,
                   distance_moved)
----------------------------------
Argumentos:
    action:
        Action executada.

    dangers:
        Dicionario com perigos.

    collision:
        Booleano indicando colisao.

    simulation_ok:
        Booleano indicando se robot.step() nao retornou -1.

    distance_moved:
        Distancia percorrida pelo robo no ultimo step.

O que faz:
    Centraliza a criacao do dicionario info retornado por step().

Retorno:
    Dicionario info.


Metodo: _restore_robot_pose(self, random_spawn=RANDOM_SPAWN)
------------------------------------------------------------
Argumentos:
    random_spawn:
        Se True, sorteia uma pose. Se False, usa a pose inicial.

O que faz:
    Se o robo for Supervisor e tiver robot_node:
    - restaura translation e rotation iniciais quando RANDOM_SPAWN for False;
    - sorteia X, Y e orientacao em torno de Z quando RANDOM_SPAWN for True;
    - preserva a altura Z inicial do robo;
    - chama resetPhysics() no node;
    - chama simulationResetPhysics() se existir no robo.

Retorno:
    Nao retorna valor.


Metodo: _get_robot_translation(self)
------------------------------------
O que faz:
    Le a posicao atual do robo usando o campo translation do Supervisor.

Retorno:
    Lista com tres coordenadas ou None quando o node nao esta disponivel.


Metodo: _get_distance_moved(self)
---------------------------------
O que faz:
    Calcula a distancia percorrida desde o step anterior comparando translation.

Retorno:
    Float com a distancia percorrida.


Metodo: _get_observation(self)
------------------------------
Argumentos:
    Nenhum.

O que faz:
    - Chama distance_sensors.read_all().
    - Normaliza os sensores entre 0.0 e 1.0.
    - Obtem as velocidades comandadas e normalizadas das quatro rodas.
    - Obtem a ultima acao no formato one-hot.
    - Concatena sensores, velocidades e acao.

Retorno:
    np.ndarray com shape (OBSERVATION_SIZE,), atualmente com 24 valores.


Metodo: _get_dangers(self, observation)
---------------------------------------
Argumentos:
    observation:
        Vetor completo normalmente retornado por _get_observation().

O que faz:
    Considera apenas os primeiros NUM_SENSORS valores da observacao.
    Calcula os principais indicadores de perigo:
    - max: perigo maximo geral.
    - front: perigo frontal.
    - left: perigo esquerdo.
    - right: perigo direito.

Retorno:
    Dicionario:

        {
            "max": float,
            "front": float,
            "left": float,
            "right": float,
        }


Metodo: _has_collision(self, dangers)
-------------------------------------
Argumentos:
    dangers:
        Dicionario com os indicadores de perigo.

O que faz:
    Considera colisao quando:
    - perigo frontal atinge COLLISION_THRESHOLD; ou
    - qualquer sensor atinge SIDE_COLLISION_THRESHOLD.

Retorno:
    Booleano indicando colisao.


Metodo: _is_safe_spawn(self, dangers)
-------------------------------------
Argumentos:
    dangers:
        Dicionario com os indicadores de perigo.

O que faz:
    Verifica se o episodio pode comecar sem perigo frontal alto e sem colisao
    imediata.

Retorno:
    Booleano indicando se o spawn e seguro.


Metodo: _get_front_danger(self, observation)
--------------------------------------------
Argumentos:
    observation:
        Vetor normalizado de sensores.

O que faz:
    Seleciona os indices FRONT_SENSOR_IDS e retorna o maior valor.

Retorno:
    Float com perigo frontal.


Metodo: _get_left_danger(self, observation)
-------------------------------------------
Argumentos:
    observation:
        Vetor normalizado de sensores.

O que faz:
    Seleciona os indices LEFT_SENSOR_IDS e retorna o maior valor.

Retorno:
    Float com perigo esquerdo.


Metodo: _get_right_danger(self, observation)
--------------------------------------------
Argumentos:
    observation:
        Vetor normalizado de sensores.

O que faz:
    Seleciona os indices RIGHT_SENSOR_IDS e retorna o maior valor.

Retorno:
    Float com perigo direito.


Metodo: close(self)
-------------------
Argumentos:
    Nenhum.

O que faz:
    Para o robo chamando wheels.action(Action.STOP).

Retorno:
    Nao retorna valor.


ARQUIVO: base_controller.py
===========================

Objetivo
--------
Ser o ponto de entrada do controller do Webots.
Este arquivo cria o Supervisor, cria o ambiente, escolhe algoritmo, cria modelo,
opcionalmente valida o ambiente com check_env e executa o treinamento.


Variaveis de configuracao
-------------------------

ALGORITHM = "PPO"
    Define qual algoritmo usar.
    Valores aceitos:
        "PPO"
        "DQN"

TRAINING = True
    Se True, treina o modelo.
    Se False, cria o ambiente e fecha sem treinar.

CHECK_ENV = False
    Se True, chama check_env(env, warn=True).
    Util para validar compatibilidade com Gymnasium/Stable-Baselines3.

TOTAL_TIMESTEPS = 200_000
    Quantidade de timesteps usada em model.learn().

CHECKPOINT_INTERVAL = 10_000
    Intervalo usado para salvar o modelo durante o treinamento.
    Evita perder toda a rodada caso a simulacao seja interrompida.

MODEL_VERSION = "stable_reward_v1"
    Identifica checkpoints treinados com a reward baseada em deslocamento real
    reset aleatorio seguro e penalidade forte para colisao.
    Evita carregar automaticamente modelos antigos.


Classe: SaveCheckpointCallback(BaseCallback)
-------------------------------------------
Callback chamado durante o treinamento.
Salva o modelo a cada CHECKPOINT_INTERVAL passos.
Para DQN, salva tambem o replay buffer.
Tambem registra metricas customizadas no TensorBoard:
    custom/mean_step_reward
    custom/mean_distance_moved
    custom/collision_rate
    custom/action_0_rate
    custom/action_1_rate
    custom/action_2_rate
    custom/action_3_rate


Funcao: make_env(robot)
-----------------------
Assinatura:

    make_env(robot)

Argumentos:
    robot:
        Supervisor do Webots.

O que faz:
    - Cria PioneerEnv com MAX_STEPS, SENSOR_LIMIT, COLLISION_THRESHOLD e
      SIDE_COLLISION_THRESHOLD vindos de config.py.

Retorno:
    Instancia de PioneerEnv.

Como usar:

    robot = Supervisor()
    env = make_env(robot)


Funcao: make_model(env)
-----------------------
Assinatura:

    make_model(env)

Argumentos:
    env:
        Ambiente Gymnasium que sera usado pelo Stable-Baselines3.

O que faz:
    - Le ALGORITHM.
    - Procura o arquivo salvo correspondente ao algoritmo:
        "ppo_pioneer_stable_reward_v1_model.zip"
        "dqn_pioneer_stable_reward_v1_model.zip"
    - Se o arquivo existir, carrega o modelo salvo com o ambiente atual.
    - Para DQN, tambem carrega o replay buffer salvo quando ele existir.
    - Se nao existir, cria um modelo novo PPO ou DQN.

Retorno:
    Modelo Stable-Baselines3 pronto para chamar learn().
    Pode retornar uma instancia nova ou carregada de PPO ou DQN.

Erros:
    Levanta ValueError se ALGORITHM nao for "PPO" nem "DQN".

Como usar:

    model = make_model(env)
    model.learn(total_timesteps=TOTAL_TIMESTEPS)


Funcao: model_path()
--------------------
Assinatura:

    model_path()

Argumentos:
    Nenhum.

O que faz:
    Gera o nome do arquivo do modelo a partir de ALGORITHM.

Retorno:
    String:
        "ppo_pioneer_stable_reward_v1_model" se ALGORITHM = "PPO"
        "dqn_pioneer_stable_reward_v1_model" se ALGORITHM = "DQN"


Funcao: replay_buffer_path()
----------------------------
Assinatura:

    replay_buffer_path()

Argumentos:
    Nenhum.

O que faz:
    Gera o nome do arquivo usado para persistir o replay buffer do DQN.

Retorno:
    String:
        "dqn_pioneer_stable_reward_v1_replay_buffer.pkl" se ALGORITHM = "DQN"


Funcao: save_model(model, label)
--------------------------------
Argumentos:
    model:
        Modelo PPO ou DQN que deve ser salvo.

    label:
        Texto usado na mensagem exibida no console.

O que faz:
    - Salva o modelo no caminho retornado por model_path().
    - Se o modelo for DQN, salva tambem o replay buffer.
    - Exibe no console a quantidade de timesteps salva.


Funcao: main()
--------------
Assinatura:

    main()

Argumentos:
    Nenhum.

O que faz:
    - Cria robot = Supervisor().
    - Cria env = make_env(robot).
    - Se CHECK_ENV for True, chama check_env(env, warn=True).
    - Se TRAINING for False, fecha o ambiente e retorna.
    - Cria ou carrega model = make_model(env).
    - Exibe no console o ponto inicial e a quantidade adicional de timesteps.
    - Treina com model.learn(), reset_num_timesteps=False e checkpoint periodico.
    - Salva o modelo final.
    - Se o usuario interromper o treinamento, salva o estado alcancado.
    - Fecha o ambiente mesmo se ocorrer erro durante o treinamento.

Retorno:
    Nao retorna valor explicito.

Como usar:
    Este arquivo e executado pelo Webots como controller.
    Tambem pode ser executado diretamente em um ambiente que tenha as dependencias:

        python base_controller.py


FLUXO GERAL DE EXECUCAO
=======================

1. Webots executa base_controller.py.
2. main() cria o Supervisor.
3. make_env(robot) cria PioneerEnv.
4. O ambiente cria DistanceSensors e Wheels.
5. make_model(env) carrega um modelo existente ou cria PPO ou DQN.
6. model.learn() chama reset() e step() varias vezes.
7. Em cada step():
    - a acao e aplicada nas rodas;
    - Webots avanca um timestep;
    - sensores sao lidos;
    - observacao e montada com sensores, velocidades comandadas e ultima acao;
    - perigos sao calculados;
    - recompensa e calculada;
    - o resultado e retornado no formato Gymnasium.
8. Ao final, o modelo e salvo e env.close() para o robo.
